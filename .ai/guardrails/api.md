# API Guardrails

> Referenced spec: `docs/functional-requirements.md` §9.2, §9.3, Appendix A & B
> These guardrails enforce the contract-first approach, REST naming conventions, and RFC 7807 error responses across all endpoints.

---

## GR-03 · `follow_rest_naming`

| | |
|---|---|
| **Severity** | 🔴 error — halts generation and delivery |
| **Enforcement** | OpenAPI spec linting + `openapi-contract-test` after_codegen hook + Checkstyle |

### Rule

All API paths must follow these conventions (§9.3):

- **Base path**: `/api/v1/`
- **Resources**: plural nouns, kebab-case (e.g., `/api/v1/maintenance-requests`, `/api/v1/service-providers`)
- **Actions on resources**: use sub-resources, never verbs (e.g., `POST .../assignments`, `POST .../decisions`)
- **Standard HTTP methods**: GET (read), POST (create/action), PUT (full update), PATCH (partial update), DELETE (remove)
- **Path parameters**: camelCase ID names (e.g., `{id}`, `{requestId}`)

### Valid vs Invalid Examples

| ✅ Valid | ❌ Invalid | Reason |
|---------|----------|--------|
| `POST /api/v1/maintenance-requests` | `POST /api/v1/createMaintenanceRequest` | Verb in path |
| `POST /api/v1/maintenance-requests/{id}/assignments` | `POST /api/v1/maintenance-requests/{id}/assign` | Singular sub-resource |
| `POST /api/v1/maintenance-requests/{id}/decisions` | `PUT /api/v1/maintenance-requests/{id}/decide` | Verb + wrong method |
| `GET /api/v1/service-providers` | `GET /api/v1/serviceProviders` | camelCase instead of kebab |
| `GET /api/v1/maintenance-requests/{id}/inspections` | `GET /api/v1/getInspections/{id}` | Verb prefix |

### Complete Valid Endpoint List (Appendix A)

```
POST   /api/v1/maintenance-requests
GET    /api/v1/maintenance-requests
GET    /api/v1/maintenance-requests/{id}
POST   /api/v1/maintenance-requests/{id}/assignments
POST   /api/v1/maintenance-requests/{id}/inspections
GET    /api/v1/maintenance-requests/{id}/inspections
POST   /api/v1/maintenance-requests/{id}/decisions
GET    /api/v1/maintenance-requests/{id}/decisions
GET    /api/v1/events
GET    /api/v1/service-providers
```

### How It Is Checked

1. **Spectral OpenAPI linting** — `npx @stoplight/spectral-cli lint api/openapi.yaml` with a custom ruleset:
```yaml
# .spectral.yaml
rules:
  paths-kebab-case:
    description: Paths must use kebab-case
    given: "$.paths[*]~"
    severity: error
    then:
      function: pattern
      functionOptions:
        match: "^/api/v1(/[a-z][a-z0-9-]*)*(/\\{[a-zA-Z]+\\})*(/[a-z][a-z0-9-]*)*$"
  no-verb-in-path:
    description: No HTTP verbs in path segments
    given: "$.paths[*]~"
    severity: error
    then:
      function: pattern
      functionOptions:
        notMatch: "/(create|get|update|delete|fetch|submit|assign|approve|reject)/"
```

2. **Checkstyle custom rule** — scans `@RequestMapping`, `@GetMapping`, `@PostMapping` annotations for verb-prefixed paths.

3. **`review-checklist` after_codegen hook** — grep on all controller classes:
```bash
grep -rn "@RequestMapping\|@GetMapping\|@PostMapping\|@PutMapping\|@PatchMapping" src/
# Manually verify no verb segments
```

## GR-07 · `openapi_contract_first`

| | |
|---|---|
| **Severity** | 🔴 error — halts generation and delivery |
| **Enforcement** | `validate-openapi-contract` before_codegen hook + `openapi-contract-test` after_codegen hook |

### Rule

The `api/openapi.yaml` file is the **single source of truth** for all API contracts. Server-side Spring Boot controllers must implement interfaces generated by `openapi-generator-maven-plugin`. No hand-written `@RequestMapping` method signatures may diverge from the spec (§9.2).

### Required Workflow

```
1. Edit api/openapi.yaml
        │
        ▼
2. Run mvn generate-sources
        │
        ▼
3. Generated interface appears in target/generated-sources/
   (e.g., MaintenanceRequestsApi.java)
        │
        ▼
4. Controller class implements the generated interface
   @RestController
   public class MaintenanceRequestController implements MaintenanceRequestsApi { ... }
        │
        ▼
5. openapi-contract-test hook verifies implementation matches spec
```

### Prohibited Patterns

```java
// ❌ BANNED: Hand-written endpoint not matching spec
@PostMapping("/api/v1/maintenance-request")   // missing 's' — diverges from spec
public ResponseEntity<RequestDto> create(...) { }

// ❌ BANNED: Controller does NOT implement generated interface
@RestController
public class MaintenanceRequestController {   // should implement MaintenanceRequestsApi
    @PostMapping("/api/v1/maintenance-requests")
    public ResponseEntity<RequestDto> createMaintenanceRequest(...) { }
}

// ❌ BANNED: Extra endpoint not declared in openapi.yaml
@GetMapping("/api/v1/maintenance-requests/export")  // not in Appendix A
public ResponseEntity<byte[]> export() { }
```

### Required Pattern

```java
// ✅ CORRECT: Implements generated interface
@RestController
@RequiredArgsConstructor
public class MaintenanceRequestController implements MaintenanceRequestsApi {

    private final CreateMaintenanceRequestService createService;

    @Override
    public ResponseEntity<RequestDto> createMaintenanceRequest(
            @Valid @RequestBody CreateRequestDto dto) {
        return ResponseEntity.status(HttpStatus.CREATED)
                             .body(createService.create(dto));
    }
}
```

### How It Is Checked

1. **`validate-openapi-contract` before_codegen hook** — verifies `api/openapi.yaml` exists and all 10 Appendix A paths are declared before generation starts.
2. **Compile-time enforcement** — if `MaintenanceRequestController` declares a method not in `MaintenanceRequestsApi`, the Java compiler fails (`mvn compile`).
3. **`openapi-contract-test` after_codegen hook** — runs `openapi-validator` against all live endpoints, comparing responses to spec schemas.
4. **ArchUnit rule** — every class in `..controller..` must implement an interface from `..bff.api..` (the generated package):
```java
@ArchTest
static final ArchRule controllersImplementGeneratedInterfaces = classes()
    .that().resideInAPackage("..controller..")
    .and().areAnnotatedWith(RestController.class)
    .should().implement(
        resideInAPackage("..bff.api..")
    );
```

## GR-08 · `rfc7807_errors`

| | |
|---|---|
| **Severity** | 🔴 error — halts generation and delivery |
| **Enforcement** | `review-checklist` after_codegen hook + `openapi-contract-test` hook + integration tests |

### Rule

All HTTP error responses (4xx, 5xx) must use **RFC 7807 Problem Detail** format (NFR-4, §9.3). No custom error shapes, no plain string messages, no raw exception messages exposed to the client.

### RFC 7807 Structure

```json
{
  "type":     "https://fleet-maintenance.example.com/errors/validation-failed",
  "title":    "Validation Failed",
  "status":   400,
  "detail":   "estimatedCost must be >= 0",
  "instance": "/api/v1/maintenance-requests/123/inspections"
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `type` | Yes | URI identifying the error type (stable, bookmarkable) |
| `title` | Yes | Human-readable summary of the error type |
| `status` | Yes | HTTP status code (integer) |
| `detail` | No | Human-readable explanation specific to this occurrence |
| `instance` | No | URI of the specific request that caused the error |

### Required Implementation

```java
// Global exception handler — one @ControllerAdvice for all errors
@ControllerAdvice
public class GlobalExceptionHandler extends ResponseEntityExceptionHandler {

    // Validation errors (400)
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ProblemDetail> handleValidation(
            MethodArgumentNotValidException ex, HttpServletRequest request) {

        ProblemDetail problem = ProblemDetail.forStatusAndDetail(
            HttpStatus.BAD_REQUEST,
            ex.getBindingResult().getFieldErrors().stream()
              .map(e -> e.getField() + ": " + e.getDefaultMessage())
              .collect(Collectors.joining("; "))
        );
        problem.setType(URI.create("https://fleet-maintenance.example.com/errors/validation-failed"));
        problem.setTitle("Validation Failed");
        problem.setInstance(URI.create(request.getRequestURI()));
        return ResponseEntity.status(HttpStatus.BAD_REQUEST)
                             .contentType(MediaType.APPLICATION_PROBLEM_JSON)
                             .body(problem);
    }

    // State transition conflicts (422)
    @ExceptionHandler(IllegalStateTransitionException.class)
    public ResponseEntity<ProblemDetail> handleStateTransition(
            IllegalStateTransitionException ex, HttpServletRequest request) {

        ProblemDetail problem = ProblemDetail.forStatusAndDetail(
            HttpStatus.UNPROCESSABLE_ENTITY, ex.getMessage()
        );
        problem.setType(URI.create("https://fleet-maintenance.example.com/errors/invalid-state-transition"));
        problem.setTitle("Invalid State Transition");
        problem.setInstance(URI.create(request.getRequestURI()));
        return ResponseEntity.status(HttpStatus.UNPROCESSABLE_ENTITY)
                             .contentType(MediaType.APPLICATION_PROBLEM_JSON)
                             .body(problem);
    }

    // Resource not found (404)
    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<ProblemDetail> handleNotFound(
            ResourceNotFoundException ex, HttpServletRequest request) {
        ProblemDetail problem = ProblemDetail.forStatusAndDetail(HttpStatus.NOT_FOUND, ex.getMessage());
        problem.setType(URI.create("https://fleet-maintenance.example.com/errors/not-found"));
        problem.setTitle("Resource Not Found");
        problem.setInstance(URI.create(request.getRequestURI()));
        return ResponseEntity.status(HttpStatus.NOT_FOUND)
                             .contentType(MediaType.APPLICATION_PROBLEM_JSON)
                             .body(problem);
    }
}
```

### OpenAPI Spec — Reusable Error Response

```yaml
# In api/openapi.yaml
components:
  responses:
    BadRequest:
      description: Validation error (RFC 7807)
      content:
        application/problem+json:
          schema:
            $ref: '#/components/schemas/ProblemDetail'
    UnprocessableEntity:
      description: Business rule or state transition violation
      content:
        application/problem+json:
          schema:
            $ref: '#/components/schemas/ProblemDetail'
    NotFound:
      description: Resource not found
      content:
        application/problem+json:
          schema:
            $ref: '#/components/schemas/ProblemDetail'
```

### How It Is Checked

1. **`review-checklist` after_codegen hook** — asserts every `@ExceptionHandler` method returns `ResponseEntity<ProblemDetail>`, not any custom class.
2. **`openapi-contract-test` hook** — sends invalid payloads to each endpoint and asserts the response contains `type`, `title`, and `status` fields with `Content-Type: application/problem+json`.
3. **Integration test pattern**:
```java
@Test
void createRequest_withMissingVehicleId_returns400WithRFC7807() {
    given()
        .body("""{ "description": "test", "priority": "LOW" }""")
        .contentType(ContentType.JSON)
    .when()
        .post("/api/v1/maintenance-requests")
    .then()
        .statusCode(400)
        .contentType("application/problem+json")
        .body("type",   notNullValue())
        .body("title",  equalTo("Validation Failed"))
        .body("status", equalTo(400))
        .body("detail", containsString("vehicleId"));
}
```
